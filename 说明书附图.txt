图1为本发明实施例提供的方法流程图

图1描述了基于多智能体辩论的无人机群综合性能评估方法的整体流程，包括以下主要步骤：
- 步骤S1：轨迹抽象与证据表示，将原始无人机轨迹数据转换为结构化领域特定语言DSL
- 步骤S2：层次化多智能体辩论协议，根据信息论复杂度动态调整辩论深度
- 步骤S3：对抗-协作混合机制，采用动态红蓝队机制缓解回声室效应
- 步骤S4：动态角色轮换，防止视角锁定并确保全面的多方面分析
- 步骤S5：结构化通信协议，智能体使用半结构化格式进行通信
- 步骤S6：证据链可追溯性，确保每个主张由可验证的证据链支持
- 步骤S7：多维共识建模，在四个正交维度上量化共识
- 步骤S8：元认知质量监控，计算四个质量指标并触发主动干预
- 步骤S9：确定性安全验证层，并行监控状态向量对抗硬约束
- 步骤S10：最终评估报告生成，基于共识和监控结果生成最终评估报告

图2为本发明实施例提供的层次化多智能体辩论协议架构图

图2展示了三层层次化辩论协议的结构：
- 第一层：快速共识层，用于低复杂度任务（Hcomp < τ1），智能体执行单轮加权聚合
- 第二层：深度分析层，用于标准任务（τ1 ≤ Hcomp < τ2），激活迭代对抗辩论协议
- 第三层：元辩论层，用于高复杂度/异常任务（Hcomp ≥ τ2），引入预辩论"标准对齐"轮次

图中还展示了复杂度路由机制，根据Hcomp = α·|ℰ|/Tduration + β·Entropy(𝒱swarm)自动选择合适的辩论层。

图3为本发明实施例提供的对抗-协作混合机制示意图

图3展示了动态红蓝队机制的运作方式：
- 红队分配：Nred = max(1, ⌊N·ρred⌋)，其中ρred = 0.4
- 红队索引随轮次旋转：ℐred(r) = {(i + r) mod N : 0 ≤ i < Nred}
- 红队任务：主动搜索异常和潜在风险，挑战乐观分数，识别边界情况，质疑假设
- 蓝队任务：提供客观专业评估，承认有效批评，反驳不合理挑战，保持建设性立场

图中展示了三个智能体（飞行控制专家FCS、无人机群协调专家SCE、安全评估专家SAE）在不同轮次中的角色分配和轮换过程。

图4为本发明实施例提供的证据链可追溯性机制流程图

图4展示了证据链的组成和验证流程：
- 证据链结构：ℰ(c) = (d, m, s, c, γ)，包括原始数据d、处理后的指标m、推理步骤s、主张c和置信度分数γ
- 验证流程：
  1. 解析每个智能体的响应以提取主张和支持证据
  2. 验证DSL引用与实际轨迹数据
  3. 拒绝不支持的主张
- 完整性检查：Ivalid(ℛ) = 𝟙[claim ∈ ℛ] ∧ 𝟙[evidence ∈ ℛ] ∧ 𝟙[reasoning ∈ ℛ] ∧ 𝟙[γ ∈ ℛ]

图中还展示了DSL令牌的使用示例，如SEG[2]: t=15-25, dir=NE->E, v=12.5±0.5, turn=high和EVENT: t=20, sharp_turn=45deg。

图5为本发明实施例提供的多维共识建模示意图

图5展示了四个正交维度的共识计算方法：
- 分数一致性：Cscore = 1 - std({γi : i ∈ [N]})
- 语义一致性：Csemantic = (2/(N(N-1)))∑i<j Sim(Si, Sj)
- 优先级一致性：Cpriority = (2/(N(N-1)))∑i<j Sim(ci, cj)
- 关注点一致性：Cconcern = |∩i Words(ei)|/|∪i Words(ei)|

图中展示了三个智能体（FCS、SCE、SAE）在四个维度上的共识计算过程和结果聚合方式。
